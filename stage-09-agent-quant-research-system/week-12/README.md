# 第 12 周：Agent 评估体系与阶段项目

## 本周目标

- 理解 Agent 评估体系的核心概念、适用条件、工程接口、验证方法与主要风险。
- 整合任务规划、工具调用、数据检索、代码执行、结果批判、知识库和自动报告。
- 完成可审计的 Agent 量化研究原型，并形成阶段总结。

## 核心成果

交付可审计的 Agent 量化研究原型、评估报告、安全审查和阶段总结。

## 每日安排

每日学习内容与索引均直接保存在当前周目录。

## 完成标准

- 能脱离代码解释 Agent 评估目标、假设、权限边界和不适用场景。
- 输入、输出、数据时点、工具权限、配置和版本均有记录。
- 规划、执行、审查、报告和审计形成完整研究链路。
- 至少覆盖正常、边界、工具失败、错误结论和越权尝试案例。
- 新环境可依据 README 复现阶段项目和评估结果。

## 每日学习路径

| 天 | 主题 | 学习与实践 | 当天输出 |
|---:|---|---|---|
| 1 | 评估目标与边界 | 定义正确性、可复现性、成本、安全和人工接管指标 | 评估指标字典 |
