# 阶段 04：多因子组合与完整研究项目

**时间：第 10—12 个月**

## 阶段目标

将单因子研究升级为可复现的多因子组合研究，完成因子库、组合构建、成本控制、归因与样本外验证。

本阶段延续“问题定义 → 数据与接口 → 基准实现 → 集成验证 → 风险审查 → 可复现交付”的学习方式。

## 阶段核心项目

完成一个包含因子库、复合评分、组合构建、成本模型、归因和样本外验证的多因子选股项目。

## 阶段完成标准

- 阶段核心模块具有清晰接口、配置、日志和自动测试。
- 数据、代码、参数、模型、策略和报告均可追踪版本。
- 样本外结果、交易成本、风险与失败案例被完整记录。
- 新环境可以依据 README 复现主要实验或运行流程。
- 完成阶段总结，明确技术债、风险边界和下一阶段计划。

## 每周执行节奏

1. 第 1 天理解概念、目标和适用边界。
2. 第 2 天整理数据、输入、接口或约束。
3. 第 3—4 天实现基准与扩展功能。
4. 第 5 天集成现有系统并核验输出。
5. 第 6 天进行测试、稳健性、风险或故障检查。
6. 第 7 天完成周项目、复盘与 Git 提交。

## 统一工程与研究原则

- 时间、数据、参数和版本语义必须明确。
- 先实现简单基准，再引入复杂模型或自动化。
- 所有自动决策均设置限制、告警和人工接管路径。
- 评价结果同时包含效果、风险、成本、稳定性和维护复杂度。
- 关键结论必须能够从基础数据和日志重新计算。
