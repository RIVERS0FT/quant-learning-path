# 第 12 周：实盘偏差评估与阶段项目

## 本周目标

- 理解实盘偏差评估的核心概念、适用条件、工程接口、验证方法与主要风险。
- 整合研究、信号调度、订单、成交、风控、监控和复盘流程。
- 完成模拟交易与小资金实盘验证闭环，并形成阶段总结。

## 核心成果

交付研究、下单、监控和复盘闭环、偏差评估报告、运行手册与阶段总结。

## 每日安排

每日学习内容与索引均直接保存在当前周目录。

## 完成标准

- 能脱离代码解释回测、模拟和实盘之间的主要偏差来源。
- 输入、输出、数据时点、配置、版本和交易约束均有记录。
- 信号、订单、成交、账户、风险和监控形成完整闭环。
- 至少覆盖正常、边界、拒单、断流和对账失败案例。
- 新环境可依据 README 和运行手册复现阶段项目。
