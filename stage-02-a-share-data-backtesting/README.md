# 阶段 02：A 股数据处理与日频回测系统

**时间：第 4—6 个月**

## 阶段目标

在第一阶段的数据与统计基础上，建立一套面向 A 股日频研究的完整工程链路：

- 标准化行情、证券状态、交易日历和公司行为数据。
- 构建按历史时点可见的股票池与可交易状态。
- 实现信号、组合、订单、撮合、账户和绩效报告模块。
- 纳入交易成本、T+1、停牌与价格限制等约束。
- 通过参数敏感性、时间分段和执行假设测试识别过拟合。
- 形成可复现、可测试、可扩展的日频回测框架。

## 周学习路线

各周学习计划见本目录下的 `week-*` 子目录。

## 阶段核心项目

完成一个从原始数据开始、能够自动生成研究报告的 A 股日频回测项目。项目至少包含：

1. 数据采集与标准化。
2. 证券主表、交易日历与公司行为。
3. 历史股票池与每日可交易状态。
4. 信号、组合、订单、撮合和账户模块。
5. 佣金、税费、滑点与约束模型。
6. 绩效分析、稳健性测试和自动报告。
7. 单元测试、集成测试、配置文件与复现说明。

## 阶段完成标准

- 原始数据、标准数据和研究结果可以追溯。
- 回测严格区分信号时点、下单时点、成交时点和估值时点。
- 现金、持仓市值与总资产能够逐日对账。
- 交易限制与成本通过集中配置管理。
- 核心模块具有自动测试。
- 能解释结果中的收益来源、交易成本、主要风险和潜在偏差。

## 每周执行节奏

1. 前两天学习概念、检查数据和定义接口。
2. 中间三天实现核心功能并构造测试用例。
3. 第六天完成数据核验、对账或敏感性测试。
4. 第七天整合本周模块、编写文档并提交 Git。

## 统一研究原则

- 所有数据转换保留来源、版本和处理日志。
- 所有信号只使用当时可见的信息。
- 所有回测明确价格口径、执行时间、费用和交易限制。
- 所有指标都能从基础流水表重新计算。
- 不只保存最优实验，同时保存失败实验与异常结果。
