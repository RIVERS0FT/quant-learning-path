# 第 7 周：回测系统架构

## 本周目标

- 区分向量化回测与事件驱动回测的适用范围。
- 定义数据、策略、组合、撮合、账户和报告模块边界。
- 建立可测试、可替换的回测接口。

## 核心成果

完成日频回测系统设计文档和最小可运行框架。

## 每日安排

每日学习内容与索引均直接保存在当前周目录。

## 完成标准

- 数据流与时间顺序清晰。
- 模块之间通过明确对象或表结构交互。
- 最小回测可在固定样本上重复运行。
