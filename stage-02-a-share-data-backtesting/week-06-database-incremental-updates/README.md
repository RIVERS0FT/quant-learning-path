# 第 6 周：数据库设计与增量更新

## 本周目标

- 设计原始层、标准层和研究层数据分层。
- 使用 DuckDB 与 Parquet 支持高效的日线查询。
- 实现幂等的增量下载、校验和版本记录。

## 核心成果

建立可更新的本地 A 股数据仓库。

## 每日安排

每日学习内容与索引均直接保存在当前周目录。

## 完成标准

- 重复运行更新任务不会生成重复数据。
- 分区和排序适合按日期与股票查询。
- 每次更新都有日志、版本和质量结果。

## 每日学习路径

| 天 | 主题 | 学习与实践 | 当天输出 |
|---:|---|---|---|
| 1 | 数据分层 | 设计 raw、clean、feature 和 metadata 层 | 数据架构图 |
| 2 | 表与文件布局 | 确定行情、证券、公司行为和日历的存储结构 | 目录与表设计 |
| 3 | Parquet 分区 | 比较按年份、月份和交易所分区的权衡 | 分区方案 |
| 4 | DuckDB 查询 | 编写筛选、连接、窗口和批量读取查询 | 查询脚本 |
| 5 | 增量更新 | 设计水位线、补数、重试和幂等写入 | 更新任务 |
| 6 | 版本与校验 | 记录批次、哈希、行数和质量指标 | 数据版本表 |
| 7 | 周项目 | 执行一次增量更新并生成更新报告 | 本地行情数据库 |
