# 第 12 周：参数敏感性、稳健性测试与阶段项目

## 本周目标

- 识别参数调优、样本选择和数据处理带来的过拟合。
- 开展时间分段、参数网格、成本压力和执行敏感性测试。
- 整合数据仓库、交易状态、回测、成本和报告模块。
- 完成一个从原始数据到研究结论的可复现策略项目。
- 补齐测试、配置、文档和运行说明。

## 核心成果

交付稳健性测试框架、策略审查报告，以及可复现、可测试、可扩展的 A 股日频回测框架。

## 每日安排

每日学习内容与索引均直接保存在当前周目录。

## 完成标准

- 测试区间与参数选择在运行前确定。
- 结果不只展示最优参数，失败实验和异常结果同样被记录。
- 新环境可按 README 完成安装和运行。
- 核心逻辑拥有单元测试与集成测试。
- 数据、配置、实验和报告能够追溯到明确版本。

## 每日学习路径

| 天 | 主题 | 学习与实践 | 当天输出 |
|---:|---|---|---|
| 1 | 偏差清单 | 复盘前视、幸存者、数据窥探和选择偏差 | 回测偏差清单 |
| 2 | 时间分段 | 比较不同市场阶段、年份和行情环境表现 | 分段绩效表 |
| 3 | 参数网格 | 绘制参数与绩效关系而非只取最优点 | 参数热力数据 |
| 4 | 成本压力 | 提高费用、滑点与成交限制测试策略韧性 | 压力测试结果 |
| 5 | 执行敏感性 | 改变信号延迟、成交价和调仓频率 | 执行敏感性报告 |
| 6 | 扰动与集成测试 | 对股票池、日期和信号进行扰动，并完成端到端对账 | 扰动结果与集成测试报告 |
| 7 | 阶段项目交付 | 整理代码、配置、测试、报告和复现说明，形成继续研究、修改或淘汰结论 | A 股日频回测框架与阶段总结 |
